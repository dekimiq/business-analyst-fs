**Пайплайн:**


**Шаблон сервиса интеграции:**

```typescript
// apps/backend/app/services/integrations/yandex_direct_service.ts
import axios from 'axios'
import { z } from 'zod'
import { env } from '@project/env'
import type { YandexCampaign } from '@project/shared'

  status: z.enum(['ACTIVE', 'PAUSED', 'ENDED']),
})

export class YandexDirectService {
  private readonly baseUrl = 'https://api.direct.yandex.com/json/v5'

  // Получить список кампаний за период
  async getCampaigns(dateFrom: string, dateTo: string): Promise<YandexCampaign[]> {
    const response = await axios.post(
      `${this.baseUrl}/campaigns`,
      {
        /* тело запроса */
      },
      { headers: { Authorization: `Bearer ${env.YANDEX_TOKEN}` } },
    )

    // Валидируем внешние данные перед использованием
```

## 6. Правила кода (строгие)

### Обязательно

- `import { env } from '@project/env'` — единственный способ получить env-переменные
- `import type { ... } from '@project/shared'` — для общих типов
- Возвращаемый тип у каждой `async function` — явно
- `zod.parse()` или `zod.safeParse()` — для любых внешних данных
- Комментарии над функциями и сложной логикой — на русском
- Ранний возврат через guard clauses — вместо вложенных `if`

### Запрещено

```typescript
// ❌ Точка с запятой
const x = 1;

// ❌ any
const data: any = response

// ❌ process.env напрямую
const token = process.env.YANDEX_TOKEN

// ❌ Хардкод значений
const url = 'http://localhost:3333'

// ❌ Заглушки линтера
// eslint-disable-next-line
// @ts-ignore

// ❌ as-каст без валидации
const user = data as User

// ❌ Логирование секретов
console.log('token:', env.YANDEX_TOKEN)

// ❌ Бизнес-логика в контроллере
async index({ response }: HttpContext) {
  const leads = await Lead.query().where(...).preload(...) // ← это в сервис
  return response.ok(leads)
}
```

### Форматирование

- Одинарные кавычки `'` — всегда
- Без точки с запятой `;` — всегда
- Trailing comma — всегда
- `printWidth` 100 символов — не превышать

## 7. Коммуникация с разработчиком

**Когда уточнять (перед кодом):**

- Задача затрагивает 2+ модуля
- Нет понимания какую схему БД использовать
- Неясно это новая таблица или изменение существующей
- Не ясен формат ответа API

**Формат уточнения:** один вопрос, конкретный, с вариантами ответа если возможно.

```
// ✅ Хороший вопрос агента
Эндпоинт /api/leads/stats — данные берём из схемы `backend.leads`
или нужна агрегация из нескольких таблиц?

// ❌ Плохой вопрос агента
Расскажи подробнее что нужно сделать?
```

**После выполнения задачи агент сообщает (генерирует отчет) в executions:**

1. Какие файлы созданы / изменены
2. Нужна ли миграция (`node ace migration:run`)
3. Есть ли новые env-переменные для добавления в `.env`
