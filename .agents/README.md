# Практические промпты для работы с правилами

Вот как ваши новые правила будут работать с повседневными задачами:

## Создание новой фичи во фронтенде:

-@web /tdd-flow Создать страницу профиля пользователя с формой редактирования имени и email. (Где /tdd-flow — это ваш другой Workflow, который реализует цикл "тест-код-рефакторинг" ).

## Рефакторинг бэкенда:

-@api Выдели логику отправки email из контроллера регистрации в отдельный сервис вapps/api/src/services. Создай план в артефакте.

## Поиск и исправление бага:

-@web Посмотри на этот стектрейс ошибки из консоли браузера. Найди проблемный компонент в apps/web и предложи фикс.

# Примеры с мультиагентами

```md
"Запусти две параллельные задачи:

1. Агент-1: Разработай API эндпоинт GET /api/users в сервисе apps/api. Используй правила из backend-rules.md. После завершения, создай артефакт с планом и кодом.
2. Агент-2: В сервисе apps/web создай UI компонент UserList, который будет получать и отображать данные с этого нового эндпоинта. Используй правила из frontend-rules.md. Следуй TDD-подходу: сначала напиши Playwright тест.

После того как оба агента закончат, Агент-3 (ревьюер) проверь, правильно ли web компонент вызывает api эндпоинт и соответствует ли код нашим правилам."
```
